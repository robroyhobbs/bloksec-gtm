## The Frontline Worker Gap

Traditional MFA assumes everyone has a smartphone. That assumption fails for 70% of the global workforce.

**The numbers:**
- **2.7 billion** frontline workers globally
- **70%** of the global workforce is deskless/frontline
- **80%** of deskless workers don't have company-issued devices
- **60M+** frontline workers in North America alone

**The cost of the current problem:**
- Password resets: **$70 each** (Forrester)
- Help desk calls from passwords: **40%** of all tickets (Gartner)
- Self-service reset savings: **$48K/year** average (Specops)
- Breaches from credentials: **81%** (Verizon DBIR)
- Average breach cost: **$4.8M** (IBM, 2025)

**Current workarounds (all broken):**
- Shared passwords → no accountability, compliance failure
- Proximity cards alone → single factor, not MFA
- PIN-only → weak, shoulder-surfable
- Paper logbooks → unscalable, audit nightmare
- "Skip MFA for frontline" → compliance time bomb

## Regulatory Tailwinds

Every major framework now requires phishing-resistant MFA:

| Framework | What Changed | Status |
|-----------|-------------|--------|
| PCI DSS 4.0 | Universal MFA for ALL cardholder data access | Active now |
| FDA 21 CFR Part 11 | Unique electronic signatures, 2-factor | Active now |
| CISA / OMB M-22-09 | Federal Zero Trust, phishing-resistant MFA | Active now |
| NIST SP 800-63B | AAL2/AAL3 require cryptographic authenticators | Active now |
| SOC 2 | Individual authentication, unique user IDs, MFA | Active now |
| NERC CIP | MFA + PKI for critical infrastructure | Active now |
| HIPAA Security Rule | MFA now mandatory (first update in 20 years) | Late 2026-2027 |

Compliance is no longer a differentiator — it's a forcing function.
