# Homepage Rewrite Brief

## Goal
Rewrite the BlokSec homepage so it clearly speaks to SMB and mid-market buyers with frontline workers, shared devices, compliance pressure, and recurring password-reset pain.

## What the current homepage gets wrong
- It is too broad and reads like a generic passwordless vendor page.
- It does not lead with the strongest wedge: frontline and shared-device authentication.
- It does not clearly separate office-worker and frontline-worker use cases.
- It does not make the conversion path obvious.
- It underplays the practical pain of resets, lockouts, worker switching, and rapid re-provisioning.

## What the new homepage must do
1. Make the frontline/shared-device problem obvious in the first screen.
2. Show that BlokSec covers both office and frontline workers.
3. Translate value into buyer language: fewer resets, faster worker switching, better auditability, stronger phishing resistance.
4. Show where BlokSec fits with existing stacks such as Microsoft, Google, and Okta.
5. Route visitors by use case, industry, and partner interest.
6. End with a single primary CTA and a clear secondary CTA.

## Recommended hero
### Headline
Passwordless authentication for the workers traditional MFA leaves behind.

### Subhead
BlokSec helps SMB and mid-market teams secure both office and frontline workers with fast, phishing-resistant authentication for shared devices, shift-based work, and high-turnover environments.

### Primary CTA
Book a tailored demo

### Secondary CTA
See frontline use cases

## Core message pillars
### 1. Secure every worker
BlokKey for workers with phones. BlokBadge for workers without them. One platform across office and floor.

### 2. Reduce reset and lockout burden
Lead with eliminated password reset friction, lower help-desk burden, and easier onboarding/offboarding.

### 3. Make shared-device access auditable
Emphasize worker accountability, faster sign-in, and cleaner audit trails where credentials are often shared today.

### 4. Fit existing identity stacks
Position BlokSec as a layer that works with Microsoft Entra ID, Google Workspace, Okta, Ping, and similar environments.

## Suggested section order
1. Hero
2. Why frontline and shared-device access is still broken
3. Two products, one workforce story
4. Outcome blocks: fewer resets, faster sign-in, better accountability, stronger compliance posture
5. Industry paths: manufacturing, retail, logistics, food processing, care settings
6. Integrations and stack fit
7. Proof: case studies, metrics, release maturity, docs
8. CTA block

## Copy rules
- Lead with workflow pain before technical features.
- Use plain language over IAM jargon.
- Avoid unsupported superlatives.
- Avoid saying BlokSec is the only option unless the claim is fully defensible.
- Use competitive contrast carefully, especially with Microsoft Entra.

## Proof to add
- one strong outcome per case study
- one short explanation of how rapid re-provisioning works
- one proof point around reporting/audit readiness from v4.0
- one proof point around deployment speed or onboarding simplicity

## Open questions
- Is the primary conversion action demo, trial, guided pilot, or partner conversation?
- What customer-proof metrics can be made public right now?
- Which vertical should be the default homepage emphasis first: manufacturing or retail?
