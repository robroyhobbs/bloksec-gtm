05
Cryptographic Foundations

Reduced Server-Side Secret Exposure

Three layers of cryptographic protection are designed to reduce centralized credential exposure and lower server-side risk.

ECDSA Asymmetric Crypto

  • No shared secrets between client and server
  • Private keys never transmitted over network
  • Eliminates MITM and replay attacks
  • Server compromise yields nothing usable

Split-Key Security

BlokBadge

  • Badge: encrypted partial key (Fragment A)
  • Server: complementary key (Fragment B)
  • Both fragments required for authentication
  • Lost badge = useless without server half

HD Wallet Isolation

BlokKey

  • Unique key pair generated per application
  • Hierarchical Deterministic derivation
  • Compromise of one app ≠ compromise of others
  • Keys derived on demand, not stored in bulk
